# 🛒 电商项目学习计划

## 项目概述

做一个电商系统，通过边做项目边学习的方式，覆盖 Java 后端面试的核心考点。

### 为什么选电商项目

电商是后端面试的"万能话题"，几乎所有高频考点都能在电商场景中找到：

| 面试考点 | 电商场景 |
|----------|----------|
| 高并发 | 秒杀/抢购 |
| 数据一致性 | 库存扣减 + 订单创建 |
| 分布式锁 | 多人同时买同一件商品 |
| 缓存 | 商品详情页缓存 |
| 消息队列 | 订单超时取消、发货通知 |
| 搜索 | 商品搜索、筛选 |
| 幂等性 | 重复提交订单 |

---

## 阶段一：核心业务（预计 1-2 周）

### 目标
跑通"浏览→加购→下单→支付"主流程

### 模块

- 👤 **用户模块**：注册、登录、地址管理
- 📦 **商品模块**：分类、SKU、详情
- 🛒 **购物车**：添加、修改、删除、全选
- 📋 **订单模块**：创建订单、订单列表、订单详情、取消订单
- 💰 **支付模块**：模拟支付（先做假接口）

### 面试考点

- RESTful API 设计规范
- 数据库设计（多表关联、外键 vs 冗余字段）
- 事务管理（@Transactional 在订单创建中的应用）
- 统一响应格式（Result<T>）
- 全局异常处理（@RestControllerAdvice）
- 参数校验（@Valid、自定义 Validator）

### 数据库表

```
user            用户表
address         收货地址
category        商品分类
product         商品表（名称、价格、库存、详情）
cart_item       购物车明细
order           订单主表（总金额、状态、支付方式）
order_item      订单明细
payment         支付记录
```

## 阶段三：性能与并发（预计 2 周）⭐ 重点

### 目标
解决电商核心难题

### 内容

- ⚡ **商品缓存**：Redis 缓存商品详情，解决读多写少
- ⚡ **秒杀功能**：库存预扣减 + 队列异步下单
- ⚡ **分布式锁**：防止超卖（Redisson）
- ⚡ **缓存一致性**：更新商品后同步刷新缓存
- ⚡ **缓存问题**：穿透（布隆过滤器）、击穿（互斥锁）、雪崩（随机过期时间）

### 面试考点

- Redis 数据结构与应用场景
- 缓存一致性策略（Cache-Aside / 先删缓存后改DB）
- 分布式锁的实现与看门狗机制
- 秒杀方案的演进（从简单到复杂）
- 数据库乐观锁 vs 悲观锁
- 缓存穿透/击穿/雪崩解决方案

## 阶段四：异步与消息（预计 1 周）

### 目标
订单生命周期管理

### 内容

- 📨 订单超时自动取消（延迟队列 / Redis 过期监听）

- 消息可靠性保证（ACK、重试、死信队列）
- 消息幂等性
- 异步解耦的价值

## 面试准备配合

学每个阶段时，同步整理：

1. **手写八股文**：每个知识点对应 3-5 个面试问题
2. **项目亮点总结**：每个阶段提炼 1-2 个"我能解决什么问题"
3. **GitHub 仓库**：代码 + README（项目介绍 + 技术栈 + 架构图）

## 面试"故事线"示例

> "我做了一个电商系统，核心流程是用户浏览商品、加入购物车、下单支付。
>
> 在**订单创建**环节，我用到了**事务管理**保证库存扣减和订单创建的原子性。
>
> 针对**秒杀超卖**问题，我先用了数据库乐观锁，后来引入了 **Redis 预扣减 + 分布式锁**的方案，QPS 从几百提升到了几千。
>
> 商品详情页因为**读多写少**，我用了 Redis 缓存，并解决了缓存穿透和一致性问题。
>
> 订单超时取消我用了 **RabbitMQ 延迟队列**来实现..."
